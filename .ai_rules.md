# AI Development Guidelines

This file contains project-specific rules that should be followed for all code modifications.

## Formatting & Organization
1. **libs.versions.toml**: All declarations within `[versions]`, `[libraries]`, and `[plugins]` must be sorted alphabetically by their key.
2. **build.gradle.kts**: Dependencies within the `dependencies` block must be sorted alphabetically by the dependency key, with `files()` dependencies grouped first at the top of the block.

## Documentation
1. **Class Comments**: Every class must have exactly one KDoc/JDoc comment attached to it that describes its purpose in one concise sentence. Nameless `companion object`s are exempt from this rule.
2. **Comment Persistence**: Do not add any other comments (inline, method, etc.) unless specifically requested. Do not remove or modify comments added manually by the user.
